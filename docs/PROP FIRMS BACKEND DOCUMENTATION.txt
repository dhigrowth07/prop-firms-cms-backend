PROP FIRMS & FUTURES FIRMS PLATFORM
COMPLETE BACKEND + ADMIN CMS DOCUMENTATION
(INCLUDES FUTURES, COUPONS, PLATFORMS, BROKERS, PAYOUTS, CMS)


VERSION: FINAL
INTENT: INFORMATIONAL / COMPARISON PLATFORM
SCOPE: DISPLAY-ONLY DATA (NO PAYMENTS, NO VALIDATION)


1. CORE PHILOSOPHY


This platform is:
- A DATA DISPLAY SYSTEM
- A COMPARISON & DISCOVERY TOOL
- CMS-DRIVEN
- SAFE (no financial logic)


The backend:
- Stores structured data
- Exposes read-only APIs
- Filters expired / inactive content


The CMS:
- Is the SINGLE SOURCE OF TRUTH
- Manages ALL visible data
- Requires NO developer intervention for updates


Frontend:
- NEVER calculates
- NEVER validates
- ONLY renders backend responses


2. FIRM TYPES


ENUM: firm_type
---------------
- prop_firm        (Forex / CFD firms)
- futures_firm     (Futures-only firms)


Both types exist in ONE firms table.

- end_date
- is_active
- created_at
- updated_at


RELATION:
- firm_coupons (firm_id, coupon_id)


OPTIONAL:
- coupon_account_types (coupon_id, account_type_id)


BACKEND FILTER RULE:
- is_active = true
- start_date <= NOW() OR NULL
- end_date > NOW() OR NULL


9. ADMIN CMS – FUNCTIONALITY (CRITICAL SECTION)


The CMS is REQUIRED.
Without it, the platform is not maintainable.


-----------------------------------------------------------------------
9.1 CMS ROLES
-----------------------------------------------------------------------


ADMIN
- Full access
- Can create, edit, delete everything
- Can publish/unpublish firms


EDITOR
- Can edit content
- Cannot delete firms
- Cannot manage system settings


-----------------------------------------------------------------------
9.2 CMS GLOBAL SECTIONS
-----------------------------------------------------------------------


GLOBAL DATA MANAGEMENT
- Trading Platforms
- Brokers / Data Feeds
- Payout Methods
- Payment Methods

Each global item supports:
- Name

11. WHAT THIS SYSTEM NEVER DOES




❌ Payments
❌ Checkout
❌ Coupon validation
❌ Affiliate tracking
❌ Broker integrations
❌ Trading connections
❌ User accounts (for now)


12. FINAL GOLDEN RULES


1. If it appears in the UI → it must be in the DB
2. If it has an icon → it must be a reference table
3. If admins need to change it → it must be in CMS
4. Frontend never decides business rules
5. Backend never applies financial logic
